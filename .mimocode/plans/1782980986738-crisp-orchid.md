# Plan: PostgreSQL via Prisma + API Routes + Translatable UI

## Context

The project is a Next.js 16 + Tailwind 4 + Framer Motion laptop catalog app. It currently uses static JSON data (`data/laptops.json`) with no database, no API routes, and no functional i18n. Theme toggle and LanguageSelector UI components already exist in the Navbar. The goal is to add a PostgreSQL backend via Prisma, REST API routes, wire the catalog to live data, and make the Hero/Footer translatable.

---

## Step 1: Install Dependencies

```
npm install prisma @prisma/client
npx prisma init
```

This creates `prisma/schema.prisma` and `.env` (with `DATABASE_URL` placeholder).

**Files created:** `prisma/schema.prisma`, `.env`

---

## Step 2: Configure Prisma Schema

**File:** `prisma/schema.prisma`

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model Laptop {
  id     Int    @id @default(autoincrement())
  name   String
  rating Float
  specs  String
  price  String
  image  String
}
```

---

## Step 3: Configure .env

**File:** `.env`

```
DATABASE_URL="postgresql://user:password@localhost:5432/laptop_db?schema=public"
```

User must replace with their actual PostgreSQL credentials.

---

## Step 4: Run Migration

```bash
npx prisma migrate dev --name init
```

Creates the `laptops` table in PostgreSQL.

---

## Step 5: Create Prisma Client Singleton

**File:** `lib/prisma.ts` (new)

Standard Next.js Prisma singleton pattern to avoid multiple connections in dev:

```ts
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

export const prisma = globalForPrisma.prisma || new PrismaClient();

if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma;
```

## Step 6: Create Seed Script

**File:** `prisma/seed.ts` (new)

3 sample laptops:
1. MacBook Pro 14" M3 — rating 4.9, price "Rp 27.000.000", image placeholder
2. ASUS ROG Strix G16 — rating 4.7, price "Rp 19.000.000", image placeholder
3. Lenovo ThinkPad E14 — rating 4.6, price "Rp 12.000.000", image placeholder

`specs` stored as JSON string (e.g., `"{\"processor\":\"Apple M3\",\"ram\":\"18GB\",\"storage\":\"512GB SSD\",\"gpu\":\"Apple M3 GPU\"}"`).

Add to `package.json`:
```json
"prisma": { "seed": "npx tsx prisma/seed.ts" }
```

Run: `npx prisma db seed`

**Files modified:** `package.json`
**Files created:** `prisma/seed.ts`

## Step 7: Create API Route `/api/laptops`

**File:** `app/api/laptops/route.ts` (new)

- **GET**: Return all laptops (`prisma.laptop.findMany()`)
- **POST**: Create laptop from JSON body (`prisma.laptop.create()`)

**File:** `app/api/laptops/[id]/route.ts` (new)

- **PUT**: Update laptop by id (`prisma.laptop.update()`)
- **DELETE**: Remove laptop by id (`prisma.laptop.delete()`)

All handlers use `NextResponse` and proper error handling with status codes.
